## 0.7.9 ‚Äî 2026-04-11

### Knowledge bases

- **Dashboard crash fix** ‚Äî the Knowledge Bases page now tolerates legacy or malformed knowledge-base rows with missing counters or timestamps instead of crashing on `toLocaleString`.
- **Server-side normalization** ‚Äî knowledge-base list/detail queries now backfill missing `memoryCount`, `totalChars`, `createdAt`, and `updatedAt` values before returning data to clients.

### Installers

- **Codex TOML repair** ‚Äî the Codex installer now merges `codex_hooks = true` into an existing `[features]` table instead of appending a duplicate table that breaks `config.toml`.
- **Full install/uninstall verification sweep** ‚Äî OpenClaw, Claude Code, Codex CLI, and Factory Droid install/uninstall flows were rechecked in temp-home simulations, including safe `--purge` cleanup behavior.

## 0.7.8 ‚Äî 2026-04-11

### Integrations and installers

- **Hook-based MCP installs for Claude Code, Codex CLI, and Factory Droid** ‚Äî shared Memory Crystal hooks now install with browser-based device auth, scoped channel/session propagation, and safer host config merging.
- **Installer hardening** ‚Äî Codex / Claude / Droid hook installers preserve unrelated hooks and repair invalid Codex hook layouts instead of clobbering existing config.
- **Safe uninstall coverage** ‚Äî public uninstall scripts now exist for OpenClaw, Claude Code, Codex CLI, and Factory Droid, with safe defaults and explicit purge paths.

### Recall and memory reliability

- **Peer-scoped coach recall fixes** ‚Äî malformed scoped writes are normalized, same-peer raw message fallback is restored at session start, and cross-client leakage protections remain intact.
- **SessionStart noise reduction** ‚Äî startup hooks now emit a compact memory-active summary instead of dumping the full wake briefing and long-form instruction block.
- **Gateway-safe local store mitigation** ‚Äî local SQLite compaction is now explicit via `localStoreEnabled`, defaults off for non-opted-in runtimes, and uses WAL/file-cache guardrails when enabled.

### Knowledge bases and release workflow

- **Knowledge base imports auto-chunk raw text** ‚Äî pasted or uploaded raw text now splits automatically while preserving explicit JSON chunk arrays and delimiter-based chunk boundaries.
- **Release workflow improvements** ‚Äî docs, public assets, and branch alignment are now part of the release lane instead of an afterthought.
- **HTTP auth / MCP test refresh** ‚Äî stale API-key mock paths were updated to match the current auth contract so release audit runs are green again.

## 0.7.6 ‚Äî 2026-04-04

### Backend ‚Äî Gemini cost guardrails

- **Cron throttling** ‚Äî graph enrichment backfill reduced from 200 memories/5 min to 25 memories/hr (~96% cut). STM embedder interval widened from 5 min to 15 min. Asset embedder interval widened from 5 min to 30 min.
- **Circuit breaker** ‚Äî graph enrichment aborts after 3 consecutive Gemini failures instead of burning quota against a failing API.
- **Daily Gemini call cap** ‚Äî new `GEMINI_DAILY_CALL_CAP` env var (off by default). When set, an atomic counter in `crystalGeminiDailyUsage` blocks further Gemini calls once the cap is reached for the UTC day. Checked in graph enrichment, `embedText`, and `embedMemory`.
- **Embedding error logging** ‚Äî `embedText` now logs status code, model name, and truncated error payload. `batchEmbedTexts` returns all-null on 429/5xx to prevent amplification; falls back to individual calls on other errors.

### Backend ‚Äî Unified CRYSTAL_API_KEY

- **Single API key env var** ‚Äî `CRYSTAL_API_KEY` is now the primary env var for Gemini access. `GEMINI_API_KEY` remains as a backward-compatible fallback. All 12+ consumers (Convex actions, MCP server, plugin, scripts) updated to resolve `CRYSTAL_API_KEY ?? GEMINI_API_KEY`.
- **Updated .env.example** ‚Äî documents `CRYSTAL_API_KEY` as the preferred key alongside `GEMINI_EMBEDDING_MODEL`.

### Backend ‚Äî Tier-aware Gemini controls

- **Per-tier Gemini policy** ‚Äî `GeminiTierConfig` added to `shared/tierLimits.ts`. Free=no managed Gemini (cap 0), Pro/Starter=managed with 500 calls/day cap, Ultra/Unlimited=managed unlimited with BYOK support.
- **Per-user guardrail** ‚Äî `geminiGuardrail.incrementAndCheck` now accepts `userId`, resolves the user's tier, and enforces the tier-specific daily cap. Global `GEMINI_DAILY_CALL_CAP` env var still works as an override (lowest cap wins).
- **BYOK for Ultra** ‚Äî new `geminiApiKey` and `geminiDailyCap` fields in `organicTickState` schema. Ultra users can supply their own key and set a custom daily cap.

### Plugin ‚Äî Peer-scoped recall fixes

- **Replaced ad-hoc channel filter** with `isNonKnowledgeBaseMemoryVisibleInChannel` for consistent peer-scoped recall isolation.
- **Blocked bare-prefix memory leakage** ‚Äî memories stored under a bare channel prefix (e.g. `agent:`) no longer leak into peer-specific channels (`agent:user-1`).
- **Surface agent-prefix memories** ‚Äî memories stored with the agent's own prefix are now correctly returned in peer channel recalls.
- **Exclude global memories from scoped channels** ‚Äî unscoped memories no longer appear in peer-scoped recall results.

### Installer ‚Äî update.sh sync

- **PLUGIN_FILES list reconciled** ‚Äî removed stale `openclaw-hook.json` entry from `scripts/update.sh` to match `plugin/update.sh`.

## 0.7.4 ‚Äî 2026-04-03

### Plugin ‚Äî Memory leak fixes

- **intentCache TTL enforcement** ‚Äî `intentCache` now enforces a 30-minute TTL at read time. Stale intents are deleted and treated as absent rather than persisting indefinitely across long sessions.
- **sessionRecallCache stale eviction** ‚Äî When the 4-hour recall cache TTL expires, stale entries are now actively deleted from both `sessionRecallCache` and `sessionRecallCacheTimestamps` instead of just skipping injection.
- **conversationPulse fetch body disposal** ‚Äî Fire-and-forget fetch now explicitly cancels the response body (`r.body?.cancel?.()`) to avoid socket lingering in Node.js.

### Installer ‚Äî migrate.sh content field fix

- **Fixed `migrate.sh` sending wrong field name** ‚Äî The import payload was sending `text` but `mcpCapture` expects `content`. Every memory import attempt since launch was silently 400ing. Fixed ‚Äî re-run `migrate.sh` to actually import your memories.

## 0.7.3 ‚Äî 2026-04-03

### Plugin ‚Äî Memory leak fix

- **`clearSessionState()` helper added** ‚Äî consolidates all per-session Map cleanup into a single function. Previously, `session_end` and `dispose()` each had incomplete, hand-rolled lists of deletes that missed several Maps (`pendingUserMessages`, `sessionConfigs`, `wakeInjectedSessions`, `seenCaptureSessions`, `intentCache`). With each session leaving behind stale entries, long-running gateways accumulated unbounded Map growth.
- **`session_end` hook updated** ‚Äî now calls `clearSessionState(sessionKey)` which clears all 13 per-session caches atomically.
- **`dispose()` updated** ‚Äî now explicitly clears all Maps including `pendingUserMessages`, `sessionConfigs`, `wakeInjectedSessions`, `seenCaptureSessions`, `intentCache`, and `reinforcementTurnCounters`.

## 0.4.2 ‚Äî 2026-03-18

### Bug Fixes
- **`ingestBatch` rename** ‚Äî context engine `ingest` method renamed to `ingestBatch` to match OpenClaw's actual API contract. The ingest hook was silently never firing because OpenClaw calls `ingestBatch()`. This is the root cause fix for context not being accumulated in the local store.
- **`tokenBudget` field name** ‚Äî `assemble` and `afterTurn` now read `payload.tokenBudget` instead of `payload.budget` to match what OpenClaw actually sends.
- **Leaf compaction threshold** ‚Äî lowered from 20,000 tokens to 4,000. Sessions were overflowing the context window before ever hitting the compaction trigger. At 4k, compaction fires after ~15-20 exchanges.
- **`assemble` tail-replacement** ‚Äî when local summaries exist, `assemble` now replaces the older portion of the raw message history with summaries and keeps only the last 6 messages raw. Previously summaries were prepended but all raw messages still passed through, so context usage never actually dropped.
- **JSON schema fix** ‚Äî removed invalid `"required": false` from `channelScope` property in `openclaw.plugin.json` (must be omitted for optional fields; was crashing gateway config validator on update).

## Deployment Notes

To deploy to production:

```bash
# Deploy Convex backend (schema + functions)
npx convex deploy

# Or for local dev:
npx convex dev

# MCP server (if running standalone):
cd mcp-server && npm run build && npm start
```

**Environment variables required:**
- `GEMINI_API_KEY` ‚Äî required for embeddings and graph enrichment (Gemini-native since v0.7.5)
- `CONVEX_URL` ‚Äî Convex deployment URL (set in mcp-server `.env`)
- `GEMINI_DAILY_CALL_CAP` ‚Äî optional daily Gemini API call limit (off by default, added in v0.7.6)
