Installation is how you connect Memory Crystal to your AI helper and choose where the memory should live.

## What this means in practice

Memory Crystal currently has two main integration models:

1. **OpenClaw Plugin** for automatic capture + recall
2. **MCP-based integrations** for Claude Code, Codex CLI, Factory Droid, Claude Desktop, and other MCP clients

## How it actually works

The installation story in this repo is spread across:

- public install scripts under `apps/web/public/`
- plugin setup scripts under `scripts/`
- MCP server READMEs and package entrypoints
- environment variable examples in `.env.example`

## Hosted Memory Crystal

The fastest path is to use the hosted Memory Crystal service and connect your client with a Memory Crystal API key.

You will need:

- a Memory Crystal API key
- a supported client
- for MCP clients, either the hosted MCP endpoint or the deprecated stdio/SSE package depending on the client

## MCP server variants

### Current path: Streamable HTTP MCP

This is the mainline MCP path now. It is documented in:

- [MCP Server](/integrations/mcp-server)

Use this for:

- Claude Code
- Codex CLI
- Factory Droid
- other MCP clients that support Streamable HTTP MCP

### Legacy path: deprecated stdio/SSE MCP package

This still exists for compatibility and is currently the documented Claude Desktop path.

Use this for:

- Claude Desktop
- any environment that still expects the older stdio/SSE package instead of the hosted Streamable HTTP endpoint
