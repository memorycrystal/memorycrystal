# Convex AGENTS

## Package Identity

- Purpose: persistent memory, recall, consolidation, decay, reflection, and knowledge-graph logic.
- Scope: Convex schema, cron jobs, HTTP routes, and query/action/mutation entrypoints.

## Setup Commands

- Deploy functions: `npm run convex:deploy`
- Open Convex dev server: `npx convex dev`
- Run a function: `npx convex run <path>`
- Health check: `npm run crystal:doctor`

## Conventions

- Keep table/field names stable under `crystal*` namespace.
- Define strict value unions for memory/store/category/source to avoid drift.
- Preserve soft-delete/archival semantics in query and recall paths.
- Prefer query/mutation/action input validation with `v.object`.

### DO / DON'T

- Edit `schema.ts`, `crons.ts`, `http.ts`, and anything in `crystal/`
- Never hand-edit `_generated/*` — regenerated by Convex tooling.

## JIT Index Hints

- Inspect crons: `rg -n "crons\\.interval|crons\\.daily" convex/crons.ts`
- Inspect graph APIs: `rg -n "export const" convex/crystal/graph.ts`

## Common Gotchas

- `vectorSearch` requires embeddings dimension matching (`3072`) — must match the embedding model output (text-embedding-3-large).
- Migration/stateful actions can be expensive; run with caution against large historical memory sets.
